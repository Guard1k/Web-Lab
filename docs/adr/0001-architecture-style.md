# ADR 0001: Вибір архітектурного стилю

* **Статус:** Прийнято
* **Контекст:** Проєкт "ToDo-система" розробляється одним розробником. Потрібно обрати між мікросервісами та монолітом.
* **Рішення:** Ми обираємо **"Модульний моноліт"**.
* **Альтернативи:** Мікросервіси були розглянуті, але вони створюють надмірну складність (мережа, розгортання) для одного розробника та простого домену.
* **Наслідки:** Весь код буде в одному репозиторії та одному застосунку, але з чітким логічним поділом на `api/`, `service/` та `domain/`. Це спрощує розробку та тестування.
