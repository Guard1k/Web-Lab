# ADR 0002: Шарова архітектура

* **Статус:** Прийнято
* **Контекст:** В рамках "модульного моноліту" (ADR-0001) потрібно визначити структуру коду.
* **Рішення:** Ми використовуємо 3-шарову архітектуру:
    * `api/`: Контролери/роути. Відповідають за HTTP-запити/відповіді, валідацію вхідних даних (DTO)[cite: 181].
    * `service/`: Бізнес-логіка (use-cases).
    * `domain/`: Сутності та їхні інваріанти (наш `Task.js`).
* **Напрямок залежностей:** `api` -> `service` -> `domain`. `api` не може викликати `domain` напряму. `service` нічого не знає про HTTP.
