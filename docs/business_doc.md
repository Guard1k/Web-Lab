# 1. Назва системи
ToDo-система

# 2. Короткий опис
Простий веб-застосунок для керування особистим списком завдань. 
Дозволяє створювати, переглядати, оновлювати та видаляти завдання.

# 3. Користувачі (Ролі)
* **Користувач** (гість): Єдина роль. Може виконувати всі дії в системі.

# 4. Основні сценарії використання
* Користувач відкриває сторінку і бачить список усіх завдань.
* Користувач вводить назву нового завдання у поле та натискає "Додати", завдання з'являється у списку.
* Користувач натискає на "Виконано" біля завдання, щоб позначити його як завершене.
* Користувач натискає "Видалити" біля завдання, щоб прибрати його зі списку.

# 5. Функціональні вимоги (Що система робить)
* FR1: Створення нового завдання (Create).
* FR2: Відображення списку всіх завдань (Read).
* FR3: Оновлення статусу завдання (Update).
* FR4: Видалення завдання (Delete).

# 6. Нефункціональні вимоги (Якість)
* NFR1: Інтерфейс має бути простим та інтуїтивно зрозумілим.
* NFR2: Система має швидко реагувати на дії користувача (до 1 секунди).
* NFR3: Дані мають зберігатися в базі даних (SQLite).

# 7. Обмеження (Чого система НЕ буде робити)
* Система НЕ має користувацької реєстрації чи авторизації.
* Система НЕ підтримує спільну роботу над завданнями.
* Система НЕ має дедлайнів або нагадувань.
